Data Flow Architecture

üì• Data Ingestion

  1. User visits website ‚Üí ot.js script loads
  2. Tracking script collects data ‚Üí Page views, UTM parameters, user agent
  3. POST to /api/v1/collect ‚Üí FastAPI receives data
  4. Data validation & processing ‚Üí Backend validates and formats
  5. Parallel storage:
    • ClickHouse: Analytics events (high volume)
    • Redis: Live session tracking (5min TTL)

üìä Data Retrieval

  1. Dashboard request ‚Üí Frontend calls /api/dashboard/stats
  2. Multi-database queries:
    • ClickHouse: Aggregate analytics data
    • Redis: Live user counts
    • PostgreSQL: Resource metadata
  3. Data aggregation ‚Üí Backend combines results
  4. Real-time updates ‚Üí WebSocket or polling for live data

Security Architecture

üîí Privacy-First Design

  • No cookies: Session tracking via localStorage only
  • No fingerprinting: Minimal data collection
  • Data sovereignty: Everything stored locally
  • GDPR compliant: No personal data processing

üõ°Ô∏è Security Measures

  • Password hashing: bcrypt with salt
  • JWT tokens: Secure authentication
  • Input validation: Pydantic models
  • SQL injection prevention: Parameterized queries
  • CORS protection: Configured origins
  • Admin confirmation: Password required for destructive operations

Data Encryption

  • Database connections: SSL/TLS enabled
  • API communications: HTTPS in production
  • Secrets management: Environment variables

Performance Architecture

‚ö° High-Performance Design

  • ClickHouse optimization: Columnar storage, compression
  • Async operations: Non-blocking I/O with FastAPI
  • Caching layers: Redis for real-time data
  • Database indexing: Optimized queries
  • CDN ready: Static assets served efficiently

Scalability Features

  • Horizontal scaling: Stateless backend design
  • Database sharding: ClickHouse supports distributed setups
  • Load balancing: Nginx proxy configuration
  • Rate limiting: Configurable request limits
